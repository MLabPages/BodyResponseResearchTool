# 身体反応リサーチツール

カメラ映像から、消費者調査向けの身体反応指標を端末内で推定するローカルWebプロトタイプです。

## 起動

```powershell
node local-server.mjs
```

ブラウザで次を開きます。

```text
http://127.0.0.1:8000
```

## 記録できる指標

### 標準指標

- 体動量: カメラフレーム差分から推定
- 姿勢変化: MediaPipe Pose が読み込める場合に推定
- 顔向き: MediaPipe Face Mesh が読み込める場合に推定
- まばたき頻度: MediaPipe Face Mesh が読み込める場合に推定
- 目の開き比率: MediaPipe Face Mesh が読み込める場合に推定

### 探索指標

- 表情変化量
- 口の開き比率
- 口角変化の近似値
- 眉周辺変化の近似値
- 静止度
- 顔が検出されない時間
- 基準値からの体動/姿勢差分

### 品質指標

- 品質スコア
- 顔検出の有無
- 姿勢検出の有無
- 明るさ
- 基準値の有無
- マーカー: 調査中の任意タイミングをログに記録

## 基準計測

調査前に「基準計測」を押すと15秒間の平均値を保存します。以降のログには、体動量と姿勢変化について基準値との差分が含まれます。

## 保存データ

映像は保存しません。CSV/JSONに保存されるのは、時刻、被験者ID、刺激名、メモ、標準指標、探索指標、品質指標、マーカーのみです。

CSV列名は `standard_`, `exploratory_`, `quality_` の接頭辞で分類しています。

## 可視化

`visualizer.html` を開くと、測定ページから出力したCSV/JSONを読み込んで可視化できます。

- 体動量、姿勢変化、表情変化、品質スコアの時系列グラフ
- 最大体動量、最大表情変化、平均品質、マーカー数
- ログ抜粋テーブル

GitHub Pagesで公開する場合は、`index.html` と同じリポジトリ直下に `visualizer.html` も置いてください。

測定ページでも、記録後に「この場で結果を見る」を押すと、そのセッションのサマリー、時系列グラフ、ログ抜粋を確認できます。CSV/JSON保存とCSV自動回収はそのまま利用できます。

## 同意確認と撮影停止

測定画面の前に同意確認画面を表示します。同意チェック後にカメラ開始が可能になります。

通常操作は「測定開始」と「測定終了」です。「測定開始」でカメラと記録が始まり、「測定終了」で記録停止、CSV自動回収、結果表示まで進みます。

「カメラ」から前面カメラと外カメラを選択できます。体全体の動きを見たい場合は、スマホや外部カメラで外カメラを選び、被験者の全身が入る位置に設置してください。

## CSV自動回収

GitHub Pagesだけではデータ保存先を持てないため、Google Apps Script、Firebase、Supabaseなどの受け口URLを使います。

測定画面の「CSV自動回収」に回収先URLを入力し、「記録停止時に自動送信」をオンにすると、記録停止時に推定指標データが送信されます。映像は送信しません。

Google Apps Scriptを使う場合は、`google-apps-script-collector.gs` の内容をスプレッドシートに紐づいたApps Scriptへ貼り付け、ウェブアプリとしてデプロイします。発行されたURLを測定画面の回収先URLに入力してください。

回収先URLの例:

```text
https://script.google.com/macros/s/xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx/exec
```

Apps Scriptのデプロイでは、種類を「ウェブアプリ」にし、実行ユーザーを自分、アクセス権を「全員」または「リンクを知っている全員」にします。発行された `/exec` で終わるURLを入力します。

## 注意

姿勢推定と顔推定はCDNからMediaPipeを読み込むため、インターネット接続が必要です。CDNに接続できない場合でも、フレーム差分による動き量は動作します。
